💡 Key Learnings
🔹 Git is a Distributed Version Control System (DVCS).
 🔹 GitHub is a cloud platform used to host Git repositories.
 🔹 Branching enables parallel development without impacting the main codebase.
 🔹 Pull Requests help maintain code quality through reviews.
 🔹 GitHub Actions automates CI/CD pipelines.
 🔹 Trunk-Based Development is widely adopted in DevOps environments.
 🔹 Git Hooks help enforce quality checks before commits.

🎯 Top Git & GitHub Interview Questions
1️⃣ What is the difference between Git and GitHub?
-Git is a version control system, whereas GitHub is a cloud platform that hosts Git repositories.

2️⃣ What is the difference between Git Fetch and Git Pull?
-git fetch downloads changes only. git pull downloads and merges changes into the current branch.

3️⃣ What is Git Stash?
-A temporary storage area for uncommitted changes.

4️⃣ What is a Pull Request (PR)?
-A request to review and merge code changes into another branch.

5️⃣ What is the difference between Merge and Rebase?
-Merge creates a merge commit. Rebase creates a cleaner linear history.

6️⃣ What is HEAD in Git?
-HEAD points to the currently checked-out commit or branch.

7️⃣ What is Git Cherry-Pick?
-Used to copy a specific commit from one branch to another.

8️⃣ What are Git Hooks?
-Scripts that run automatically before or after Git events such as commits and pushes.

9️⃣ What is the difference between Git Reset and Git Revert?
-Reset removes commits from history. Revert creates a new commit that undoes previous changes.

🔟 What branching strategy is preferred in DevOps?
-Trunk-Based Development because it supports Continuous Integration and Continuous Delivery.
